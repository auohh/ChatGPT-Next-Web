## 1. 功能概述

**一句话描述**：用户可选择多个 AI 模型，对同一问题发起并行请求，在聊天界面中对比各模型的回复结果。

**用户价值**：
- **效率提升**：一次发送，多个模型同时回答，避免重复输入
- **质量对比**：直观比较不同模型的回答质量，选择最佳结果
- **成本优化**：通过对比了解各模型特点，合理选择使用
- **探索发现**：尝试同一问题在不同模型下的表现差异

## 2. 用户故事

**故事 1 - 基础对比**
> 作为 AI 重度用户，我希望能够同时向 GPT-4 和 Claude 3.5 发送同一个问题，以便快速对比哪个模型的回答更符合我的需求。

**故事 2 - 性能测试**
> 作为开发者，我希望测试同一个提示词在多个模型上的表现，以便选择最适合我业务的模型。

**故事 3 - 结果采纳**
> 作为内容创作者，我希望看到多个模型对同一话题的不同解读，以便获得更丰富的创作灵感。

**故事 4 - 最佳答案**
> 作为普通用户，我希望在看到多个回复后，能够一键采纳其中一个作为最终答案，继续后续对话。

## 3. 功能范围

### 3.1 In Scope（本次要做）

| 功能项 | 描述 |
|--------|------|
| 多模型选择 | 复用现有快捷模型选择器，支持多选模型（2-6个） |
| 并行请求 | 同时向多个模型发起流式请求 |
| 对比展示 | 网格布局展示各模型回复，支持折叠/展开 |
| 流式更新 | 各模型回复独立流式渲染，互不干扰 |
| 状态指示 | loading/streaming/error/done 状态可视化 |
| 采纳操作 | 选择最佳回复，替换为当前会话的唯一回复 |
| 复制操作 | 单独复制某个模型的回复 |
| 停止操作 | 停止所有正在进行的请求 |
| 单独停止 | 停止某个特定模型的请求 |
| 错误处理 | 某个模型失败不影响其他模型展示 |
| 元数据记录 | 记录每条回复使用的模型信息 |
| Token 统计 | 显示各回复的 token 使用情况 |

### 3.2 Out Scope（明确不做）

| 功能项 | 原因 |
|--------|------|
| 模型评分 | 需要建立评分体系，后续版本考虑 |
| 自动推荐最佳 | 需要评估算法支持，暂时由用户主观判断 |
| 历史对比记录 | 本次仅支持实时对比，历史对比后续考虑 |
| 导出对比结果 | 用户可通过复制实现，暂不提供批量导出 |
| 自定义对比维度 | 如评分项、对比维度等，暂时简化 |
| 语音输入/输出 | 与现有功能解耦，后续独立开发 |
| 图片生成模型对比 | DALL-E 等图像模型暂不支持 |

```
└─────────────────────────────────────────────────────────────────┘
```

**推荐理由**：
1. **直观对比**：网格布局便于左右/上下对比
2. **空间利用**：两列布局充分利用屏幕宽度
3. **状态清晰**：每个卡片独立显示状态
4. **响应式**：移动端自动切换为单列
5. **渐进展示**：流式内容逐卡片渲染

**列表视图（折叠模式）**：
```
┌─────────────────────────────────────────────────────────────────┐
│  ▼ GPT-4o (进行中...)                         [复制] [采纳] [停止] │
```

### 5.3 状态指示设计

| 状态 | 视觉表现 |
|------|----------|
| Loading | 卡片顶部骨架屏 + "正在连接..." |
| Streaming | 模型名称旁显示跳动的光点动画 |
| Done | 右上角绿色勾选图标 |
| Error | 红色边框 + 错误信息 + 重试按钮 |
| Stopped | 灰色边框 + "已停止"标识 |

### 5.4 操作按钮设计

| 按钮 | 位置 | 图标 | 行为 |
|------|------|------|------|
| 复制 | 卡片底部 | 📋 | 复制该回复内容，显示"已复制"提示 |
| 采纳 | 卡片底部 | ✓ | 将该回复设为正式回复，关闭对比视图 |
| 停止/重试 | 卡片底部 | ⏸️/🔄 | 停止该模型请求 / 重新发起请求 |
| 全部停止 | 顶部工具栏 | ⏹️ | 停止所有模型请求 |
| 关闭对比 | 顶部工具栏 | ✕ | 关闭对比视图，返回正常聊天 |

```typescript
  // ... 现有字段

    tokens?: number;               // token 使用量
    latency?: number;              // 响应延迟（毫秒）
  };
}
```

## 8. 边界情况与异常处理

### 8.1 边界情况

| # | 场景 | 处理方式 |
|---|------|----------|
| 1 | 选择 < 2 个模型 | 提示"至少选择 2 个模型进行对比"，禁用发送按钮 |
| 2 | 选择 > 6 个模型 | 提示"最多选择 6 个模型"，自动截断或阻止添加 |
| 3 | 所有模型都失败 | 显示"所有模型请求失败"，提供"全部重试"按钮 |
| 4 | 部分模型不可用 | 自动过滤不可用模型，提示"N 个模型不可用已跳过" |
| 5 | 用户快速切换 | 离开对比模式时自动停止所有进行中的请求 |
| 6 | 长时间未响应 | 30 秒后显示"响应较慢，请等待"，提供停止选项 |
| 7 | 对比期间切换会话 | 自动停止对比请求，状态保留 |
| 8 | 采纳后返回 | 从历史消息查看对比结果时，展示完整对比数据 |

### 8.2 异常处理

| # | 异常类型 | 错误提示 | 处理方式 |
|---|------|----------|----------|
| 1 | 网络断开 | "网络连接失败，请检查网络设置" | 显示重试按钮，保留已完成内容 |
| 2 | API Key 无效 | "API Key 无效，请前往设置" | 提供跳转设置按钮 |
| 3 | 额度耗尽 | "API 额度不足" | 显示具体错误信息 |
| 4 | 超时 | "请求超时，请稍后重试" | 显示重试按钮 |
| 5 | 内容审核 | "内容被拒绝生成" | 显示原因，允许重试 |
| 6 | 服务器错误 | "服务器暂时不可用" | 显示错误码和重试按钮 |
| 7 | 流式中断 | "连接意外断开" | 保留已接收内容，显示重试 |
| 8 | 并发限制 | "请求过多，请稍后" | 自动重试（指数退避） |

### 10.2 关键接口设计

```typescript

interface ChatStoreMethods {
  // 进入对比模式
  enableCompareMode(selectedModels: string[]): void;

  // 发起对比请求
  onUserInputWithCompare(
    content: string,
    models: string[],
    attachImages?: string[]
  ): Promise<void>;

  // 更新对比回复
  updateCompareResponse(
    sessionId: string,
    requestId: string,
    model: string,
    updater: (response: CompareResponse) => void
  ): void;

  // 采纳对比结果
  adoptCompareResult(
    sessionId: string,
    requestId: string,
    model: string
  ): void;

  // 停止对比请求
  stopCompareRequest(sessionId: string, requestId: string): void;

  // 停止单个模型
  stopCompareModel(
    sessionId: string,
    requestId: string,
    model: string
  ): void;
}
```
